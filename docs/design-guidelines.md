# デザインガイドライン

このサイトの UI を実装・変更するときは、必ずこのドキュメントに従うこと。
新しいページ・コンポーネントを足すときも、ここで定義した世界観・トークン・禁止事項を守れば
デザインが揃う。**迷ったら新しい装飾を足すのではなく、既存コンポーネントを再利用する。**

---

## 1. コンセプト

澄んだ白を基調に、淡いセージグリーンを主役に据えた、軽やかでテックなデザイン。

- 澄んだ白基調・明るく軽い空気感
- 主役は **セージグリーン**、副アクセントに **瑠璃の青**、差し色に **黄金**
- ほんのりテック / オペレーター（HUD）的な質感。ただし**抑制的に**。やりすぎない
- **ライトテーマのみ**。ダークテーマ・テーマ切替は作らない

---

## 2. 大原則

- **アクセシビリティ最優先。** 白背景に乗せるテキストは WCAG AA（4.5:1）以上、できれば AAA（7:1）。
  実装後はコントラストを実測で確認する。
- **生の Tailwind 色（`emerald-*`, `slate-*` 等）を直書きしない。** 必ず下記のデザイントークンを使う。
- **デザインの一貫性は共有コンポーネントで担保する。** 個別ページでスタイルを作り込まない。

---

## 3. カラートークン

`app/assets/css/main.css` の `@theme` で一元定義。これが唯一のソース。

| トークン | 用途 |
| --- | --- |
| `brand-50`〜`brand-900` | セージグリーン（主役）。テキストは `brand-700` 以上のみ可 |
| `azure-600` / `azure-700` | 瑠璃の青。リンク・TS比較ラベル等。テキストは `azure-700` |
| `gold-*` | 黄金（**装飾専用**）。`gold-800` のみ淡い金地の上でテキスト可 |
| `ink` | 本文（約15:1） |
| `ink-soft` | 補足文（約6.7:1） |
| `line` | カードの枠線など**境界（border）**の色。単独の区切り線には使わない（下記 divider 禁止） |
| `code-bg` | コードブロック背景（catppuccin-latte の `#eff1f5`）。インラインコードもこれに揃える |

**金（gold）はテキストに使わない**（装飾は `aria-hidden` を付ける）。唯一の例外は淡い金地の上の `gold-800`。

---

## 4. タイポグラフィ

- **普通のシステムフォントだけを使う。外部 Web フォント（Google Fonts 等）は読み込まない。**
- 本文・見出し: `--font-display` / `--font-body`（= `system-ui` 系の標準スタック）
- コード・ラベル: `--font-mono`（= `ui-monospace` 系の標準スタック）
- 装飾的なディスプレイフォントは使わない

---

## 5. コンポーネント（一貫性の担保）

| コンポーネント | 役割 |
| --- | --- |
| `CheatPage` | チートシートページの共有レイアウト（戻るリンク・ヘッダー・セクション描画） |
| `CheatSection` | セクション見出し（`◆` ＋ ラベル）＋カードのグリッド |
| `TipCard` | 1件の Tips カード（タイトル・バッジ・説明・コード） |
| `TipBadge` | 必須 / 高頻度 バッジ |
| `CodeBlock` | Shiki でハイライトしたコード |
| `CodeCompare` | TS↔Kotlin の左右比較 |
| `RichText` | 説明文中の `` `バッククォート` `` をインラインコード表示 |

**新しいチートシートの足し方**: `app/pages/<slug>.vue` を作り、データ配列＋`<CheatPage :title :description :sections>` で組む。
`index.vue` の `sheets` に1行足す。色は生クラスではなくトークンで。

---

## 6. バッジ（TipBadge）

一目で意味が伝わる色にする。

- **必須** = 赤系（重要・必ず押さえる）: `bg-red-100 text-red-700 ring-red-200`
- **高頻度** = 青系（よく出てくる・情報）: `bg-sky-100 text-sky-700 ring-sky-200`

どちらも AA を満たすこと。

---

## 7. コード表示

- ハイライトは **Shiki**、テーマは **catppuccin-latte**（ビルド時に焼き込み、クライアントへ shiki を送らない）。
- **インラインコードの背景はコードブロックと同色**（`bg-code-bg`）。文字は `text-brand-800`。
- 比較ブロックはグリッドの子に `min-w-0` を付け、コードはカード内で横スクロールさせる（はみ出し禁止）。

---

## 8. レイアウト / レスポンシブ

- チートのカードは**1列フル幅**（コード可読性のため）。比較はカード内部でグリッド分割。
- 必ずレスポンシブ対応（モバイル1列）。スマホ幅でコンテンツがはみ出さないこと。
- コンテナは読みやすい最大幅（チートシートは `max-w-4xl`、一覧は `max-w-5xl`）。

---

## 9. モーション

- ページ読み込み時の控えめなスタッガード・リビール（`.reveal`）程度に留める。
- **必ず `prefers-reduced-motion: reduce` を尊重**して無効化する。

---

## 10. 禁止事項（アンチパターン）

「AIが作ったっぽい」見た目を避ける。次は**やらない**こと:

- ❌ **グラデーションの区切り線（divider）** ― 緑→青→金のような多色グラデ罫線は使わない。
- ❌ **そもそも区切り線（divider / `hr` / ヘアライン罫線）を使わない。** セクションやヘッダーの境界は
  余白とラベルで表現する。
- ❌ **外部 Web フォント**（システムフォントを使う）。
- ❌ **詳細ページ（チートシート）のカードにホバー効果を付けない**（静止）。
  ※トップ一覧の**遷移リンク**カードはホバー可。
- ❌ 紫グラデ on 白などの定番 AI 配色。
- ❌ 生の Tailwind 色の直書き（トークンを使う）。
- ❌ 金をテキスト色に使う（装飾専用）。
